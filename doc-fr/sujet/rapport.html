Structure du rapport

Votre rapport devra adopter la structure suivante :

  1. Exécution des commandes

    Expliquez dans cette partie comment votre programme exécute les commandes. Il est important que vous utilisiez dans cette partie le vocabulaire définie dans la syntaxe. Vous décriverez donc successivement comment exécuter une commande simple composée de mots, puis une commande redirigée, puis un pipeline, etc.

    Notez qu'il s'agit d'une explication, pas du code de votre programme. Vous pouvez placer des extraits de code, mais uniquement à titre d'illustration.

    N'oubliez pas dans cette partie d'expliquer où et comment vous exécutez les commandes internes (comme cd par exemple).

  2. Gestion des signaux

    Cette partie doit décrire de quelle façon votre programme gère les signaux. En particulier, décrivez les gestionnaires de signaux que vous mettez en place, et expliquez comment vous faîtes pour « transmettre » les signaux reçus aux processus.

  3. Gestion de l'historique

    Cette partie décrit l'infrastructure nécessaire pour gérer l'historique dans une mémoire partagée. Expliquez comment chaque mini-shell utilise et modifie le contenu de la mémoire partagée, et éventuellement comment celle-ci est sauvegardé.

Vous pouvez éventuellement changer l'ordre des deux dernières parties.

Enfin, vous devrez fournir en meme temps que votre rapport le code de votre programme. Nous placerons ici dans quelques temps des indications sur les conventions à respecter.
