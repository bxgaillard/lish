Projet Systèmes d'exploitation

Sujet

(Cette partie ne concerne que les IUP GMI 2e année. Les étudiants de licence peuvent consulter https://dpt-info.u-strasbg.fr/~mathis/Systeme/Licence/tp.html.)

Ce projet consiste à écrire un mini-shell intéractif. Il s'agit donc d'un programme qui affiche un prompt, attend les commandes de la part de l'utilisateur puis lance le traitement approprié. Le mini-shell agit de la sorte en boucle jusqu'à saisir exit ou recevoir la fin de fichier Ctrl-d.

Ce mini-shell devra traiter :

  • les redirections : > < >> 2> etc.
  • les tubes : |
  • les commandes conditionnelles : && ||
  • la mise en arrière plan : &
  • les signaux Ctrl-c, Ctrl-\ qui devront être gérés (et ne pas tuer le mini-shell)
  • les commandes internes : cd, kill, exit, history (voir plus loin)

Par ailleurs, les shells partageront un historique qui sera placé dans un segment de mémoire partagée et sauvegardé dans un fichier $HOME/.history. Ce fichier sera ouvert par le premier shell lancé et mis à jour lors de la fermeture du dernier shell. Une commande interne history affichera cet historique. Il est possible d'utiliser les commandes conservées dans l'historique de la façon suivante :

  • !num : rappel de la commande de numéro num (donné par la commande history qui précéde l'affichage de chaque commande par un numéro)
  • !! : rappel de la dernière commande
  • !abc... : rappel de la dernière commande commençant par abc...

Pour faciliter votre travail, vous trouverez un parser de commandes ainsi que des explications sur son fonctionnement à la page https://dpt-info.u-strasbg.fr/~alain/systeme/. Précisons qu'il n'est pas anormal que vous passiez un peu de temps à maitriser ce parser. Le temps ainsi consacré devrait être bien inférieur à celui qu'il vous faudrait pour réaliser votre propre analyseur de commandes.

Ce projet peut être effectué en binôme. Il donnera lieu à la rédaction d'un rapport (voici quelques indications). Vous devrez également transmettre à votre responsable de TD un fichier (.tar.gz) contenant votre code.

La date limite pour rendre rapport et code est fixée au 14 janvier 2005. Cette date est impérative.

Aide

Voici les document disponibles actuellement pour votre projet :

  • Une description de la syntaxe des commandes du shell. Ce document est important parce qu'il définit le vocabulaire.

  • Une description de la bibliothèque disponible pour vous aider.

  • Une archive contenant tout ce qu'il faut pour installer la bibliothèque. Notez que vous avez besoin des outils flex et bison. Il faut aussi avoir make, et si possible gmake.

    Version courante : chelle-1.tar.gz

    Attention, l'archive contient un répertoire chelle-version et deux sous-répertoires, appelés lib/ et exemple/. Seul le premier est indispensable. Lorsque vous avez obtenu cette archive, il vous suffit de passer les commandes suivantes :

    tar xvfz chelle-version.tar.gz
make -C chelle-version/lib

    Si vous voulez tester l'exemple :

    cd chelle-version/exemple
make

    Le programme de test s'appelle chelle. Il lit les commandes que vous tapez, et affiche la structure de données correspondante.

    Attention : si vous êtes sous linux, vous pouvez utiliser directement le makefile fourni dans exemple/. Si vous êtes sous un autre système (par exemple sur ada), il faudra ajouter un # devant la ligne CFLAGS+=... dans le makefile. Ce sera modifié dans une autre version.

    Attention : n'écrivez aucun code dans le répertoire chelle-version créé par extraction de l'archive. L'analyseur de commandes contient probablement des erreurs, et il risque d'y avoir de nouvelles versions de temps en temps. Utilisez une structure de répertoire telle que :

    chelle-version/
    lib/
        commande.h
        libchelle.a
        ...
    exemple/
        chelle.c
        ...
crustace/
    votrecode1.c
    votrecode2.c
    ....

    Le répertoire de développement s'appelle crustace ici. (Shell signifie coquillage en anglais.) Ce n'est qu'un exemple.

    Supposons que REPCHELLE soit le chemin complet du répertoire crée par l'extraction de l'archive. Pour compiler vos programmes :

    gcc -W -Wall -IREPCHELLE/lib -c code.c

    Pour l'édition de liens :

    gcc -LREPCHELLE/lib code1.o code2.o ... -lchelle -o executable

    (Il est important que le paramètre -lchelle apparaisse après les fichiers .o, sinon cela ne fonctionnera pas.)

    Nous vous recommandons d'écrire un fichier makefile et de conserver le répertoire de la bibliothèque dans une variable.
